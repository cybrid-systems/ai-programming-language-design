# 02 — 编译期代码能力：std::meta + consteval

## 问题

Lisp 宏系统在编译期操作代码（宏展开器是编译器的一部分）。为了让宏展开器高性能运行，编译器需要编译期计算能力：

- 宏展开中的条件判断和控制流
- 编译期的模式匹配和树变换
- 代码生成阶段的优化决策

## std::meta 解决什么

`std::meta` 提供编译期的代码查询和生成能力：

- `std::meta::reflect()` — 反射已有的代码结构
- `std::meta::splice()` — 在编译期注入生成的代码
- 配合 `consteval`（保证在编译期执行）形编译期计算管道

## 与语言核心的对应

```
语言的宏系统（编译期 AST → AST 变换）
    ↓ 宏展开器的性能依赖编译期计算
std::meta + consteval → 编译期代码查询/生成 → 高效的宏展开引擎
```

## 核心思路

Lisp 的宏是"函数式的 AST 变换"。std::meta 是 C++26 对"在编译期操作代码"这个需求的回应。

两者思路不同（宏是代码→代码函数，std::meta 是反射+生成），但面对的同一个问题：**编译器需要在编译期理解和生成代码**。理解这一点，比比较具体 API 更重要。

`consteval` 保证函数一定在编译期执行——这对编译器的关键路径（类型检查、宏展开、代码生成）是"零运行时开销"的承诺。
