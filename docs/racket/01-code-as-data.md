# 01 — 代码即数据：AST 查询与变换

Lisp 系语言的根本特征：代码和数据是同一个东西。

## 核心

```
(+ 1 2)     ← 这是代码，也是数据（三个元素的列表）
'(+ 1 2)    ← 显式标记为数据，结构完全一样
```

没有"代码的字符串表示"，没有 parser，没有 lexer。代码就是它的 AST。

## 对 AI 意味着什么

AI 不需要"生成代码文件"，不需要"解析代码再理解"。

AI 操作的是代码的**原生结构**——通过标准的列表操作（car/cdr/cons）就可以查询、变换、生成代码。

```
;; 查询：找到函数定义中所有递归调用
(define (find-recursive-calls body)
  (filter (λ (expr) (equal? (car expr) 'self-call)) body))

;; 变换：给所有函数调用加上日志
(define (instrument-calls ast)
  (map (λ (node)
         (if (and (list? node) (symbol? (car node)))
             `(with-logging ,node)
             node))
       ast))
```

## 机制

**宏（macro）**是在编译期运行的代码→代码变换函数。它们不是"字符串替换"，而是基于**语法对象（syntax object）**的结构化操作，在编译期展开，零运行时开销。

语法对象携带：
- 代码的 AST 结构
- 源位置（错误报告）
- 作用域信息（卫生宏，自动避免名字冲突）

这保证了 AI 变换代码时不会意外捕获或暴露符号。变换是安全可组合的。

## 总结

一条原则就够了：**结构就是结构，不需要字符串这个中介。**

这就是增量编译、AST 修复、自动变换的基础——你看到的就是你能操作的。
